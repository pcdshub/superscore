superscore.backends.filestore.FilestoreBackend

class superscore.backends.filestore.FilestoreBackend(path, cfg_path=None)[source]

Bases: _Backend

Filestore configuration backend.

Holds an entry cache, filled when a file is loaded. All CRUD operations reconstruct the Root object and save a new version of the database. File storage is a json file containing serialized model dataclasses.

initialize()[source]

Initialize a new JSON file database.

Raises:
PermissionError

If the JSON file specified by path already exists.

Notes

This exists because the FilestoreBackend.store and FilestoreBackend.load methods assume that the given path already points to a readable JSON file. In order to begin filling a new database, an empty but valid JSON file is created.

store()[source]

Stash the database in the JSON file. This is a two-step process: 1. Write the database out to a temporary file 2. Move the temporary file over the previous database. Step 2 is an atomic operation, ensuring that the database does not get corrupted by an interrupted json.dump.
